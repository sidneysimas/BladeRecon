# Code of Conduct

## Our Pledge

BladeRecon is an open-source security project. We pledge to make participation
respectful, inclusive, and focused on responsible research.

## Expected Behavior

- Be respectful and constructive in issues, pull requests, and discussions.
- Keep feedback technical, actionable, and scoped to BladeRecon.
- Respect target authorization, program rules, and responsible disclosure.
- Avoid sharing live credentials, third-party private data, or sensitive target
  details in public project spaces.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Publishing private information without explicit permission.
- Encouraging unauthorized scanning, exploitation, or disclosure.
- Repeated low-effort disruption of maintainers or contributors.

## Reporting

Report conduct concerns to the project maintainer listed in `pyproject.toml`.
For security vulnerabilities in BladeRecon itself, follow `SECURITY.md`.

Include enough context for review, such as links, screenshots, timestamps, and
the people involved. Do not include sensitive third-party target data unless the
maintainer explicitly requests a private transfer method.

## Enforcement

Maintainers may remove comments, close issues, request edits, or restrict
participation for behavior that violates this code. Enforcement should be
proportionate, documented when practical, and focused on keeping the project
safe for responsible contributors.

This code is adapted from widely used open-source community standards,
including the Contributor Covenant.
